watch, digest, apply

  1. $scope.$watch() creates a watch of variable, takes 2 parameters
    • value function
    • listener function

  2. $scope.digest() iterates through all the watches in the $scope object and its child objects, When $digest() iterates over the watches, it calls the value function for each watch. If the value returned by the value function is different than the value it returned the last time it was called, the listener function for that watch is called.

  3. $scope.apply() takes a function as parameter which is executed, and after that $scope.$digest() is called internally.

Value/Constant/Factory/Service/Provider

  1. module.config() cannot inject Value, but Constant
  2. Factory is function creates values, Service is an js object
  3. Provider
    • it contains a single $get() function. This is the Factory function of the provider.
    • using module.config() by calling method inside Provider

Custom Directive

  1. compile, link, controller

    • compile should be used when you need modify directive template, it runs once and return a link
    • link is the function which used when you need to attach event handler, modify DOM.
    • controller is used when you need to share or reuse $scope data

  2. Transclusion

    • Wraps Elements, set transclude: true
    • using ng-transclude as a attribute on the tag

  3. isolating scope binding style

    • ‘@’: one way to binding with double {}
    • ‘=’: two way
    • ‘>’: one way
    • ‘&’: often used for callback function

      using scope.func()(para1, para2) to pass parameters to parent scope function
